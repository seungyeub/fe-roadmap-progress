# 📅 Study Schedule (학습 일정)

> 이 문서는 프론트엔드 로드맵 학습의 **중·장기 계획**을 정리하는 곳입니다.
> 하루 목표 학습 시간: **4시간** (상황에 따라 ±1시간 조정 가능)

---

## 1. 기본 원칙

- **월~금**: 최소 4시간 집중 학습
- **토요일**: 복습 + 프로젝트/실습 위주 (2~4시간)
- **일요일**: 휴식 또는 가벼운 복습, 주간 회고 정리
- 주 단위로 **하나의 메인 테마**에 집중
  - 예: “JS 기본 문법 정리 주간”, “TS 타입 시스템 이해 주간”, “React Hooks 집중 주간”

---

## 2. 전체 단계(Phase) 개요

| Phase | 주요 내용                              | 예상 기간 (예시) |
|-------|----------------------------------------|------------------|
| 1     | CS 기초 + JavaScript 기본/핵심 개념   | 3~4주           |
| 2     | TypeScript 기초 + React 기본/중급     | 4~5주           |
| 3     | 상태관리, 테스트, 빌드 툴, 스타일링    | 3~4주           |
| 4     | 개인/포트폴리오 프로젝트 + 면접 준비  | 4주 이상        |

> 실제 기간은 진행 상황에 따라 자유롭게 조정합니다.

---

## 3. 주차별 계획 (예시 초안)

> ⚠️ 아래는 **예시**입니다. 현재 진행 상황에 맞게 기간/주차를 수정해 주세요.

### 📌 Week 1 – CS & JS 기초 다지기 (예시: 2025-10-30 ~ 2025-11-05)

- **목표**
  - 네트워크/HTTP, 브라우저 동작 원리 개념 복습
  - JavaScript 기본 문법과 실행 환경 감각 되찾기
- **세부 계획**
  - CS: 네트워크 기본, HTTP 요청/응답, 쿠키/세션/캐시 개념 잡기
  - JS: 변수/스코프, 함수, 기본 타입, 조건문/반복문, 배열/객체
- **연결되는 Blueprint 섹션 (예시)**
  - `01_CS_Basics/network.md`
  - `02_JS/basics.md`

---

### 📌 Week 2 – JS 핵심 개념 (스코프, this, 클로저, 비동기)

- **목표**
  - 스코프/실행 컨텍스트/클로저에 대한 개념을 정확히 이해
  - 콜백, Promise, async/await 흐름 이해
- **세부 계획**
  - 월~화: 스코프, 호이스팅, 클로저 정리 + 예제
  - 수~목: this 바인딩, call/apply/bind, 화살표 함수
  - 금~토: 비동기 프로그래밍(Promise, async/await) 연습
- **연결되는 Blueprint 섹션 (예시)**
  - `02_JS/scope-closure-this.md`
  - `02_JS/async.md`

---

### 📌 Week 3 – TypeScript 기초

- **목표**
  - TS 기본 타입, 인터페이스, 타입 별칭, 제네릭 기초 이해
  - 기존 JS 코드를 TS로 점진적으로 옮길 수 있는 수준
- **세부 계획**
  - TS 프로젝트 세팅(Vite + React + TS)
  - 타입 시스템 기초 정리 + 간단한 실습
- **연결되는 Blueprint 섹션 (예시)**
  - `03_TS/basics.md`

---

### 📌 Week 4 – React 기본 구조 & Hooks

- **목표**
  - React 컴포넌트, 상태/props, JSX, 이벤트 처리 재정리
  - 주요 Hooks(useState, useEffect, useMemo, useCallback) 사용에 익숙해지기
- **세부 계획**
  - 간단한 Todo 앱, 리스트 렌더링, 폼 관리 연습
- **연결되는 Blueprint 섹션 (예시)**
  - `04_React/basics.md`
  - `04_React/hooks.md`

## 5. 실제 사용 팁

- 현실과 계획이 안 맞는 것은 **정상**이므로, 주말/주초에 `study_schedule.md`를 열고 계획을 조금씩 수정.
- 이미 지난 주차는
  - "목표 달성 여부"를 간단히 체크하고
  - 중요한 변화가 있으면 `progress_table.md`와 `weekly_summary`에도 반영.
